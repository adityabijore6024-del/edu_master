# 🚀 EduPeak — Premium EdTech Platform

A complete, production-ready EdTech platform built with **Django**, **HTML/CSS/JS**, and **Bootstrap Icons**. Inspired by Udemy, Unacademy, Coursera, Physics Wallah, and Coding Ninjas.

---

## ✨ Features

- 🎨 Premium glassmorphism UI with smooth animations, gradients & dark mode
- 🔐 Secure custom authentication (email-based, UUID PKs, hashed passwords)
- 📚 Full course system — categories, teachers, sections, lectures, ratings
- 💳 Razorpay payment integration (signature-verified)
- 🤖 AI Doubt Solver powered by the Anthropic Claude API
- 📝 Complete test engine — single/multiple/integer questions, live timer, auto-submit, unlimited attempts, question navigator
- 📊 Result dashboard with score rings & analytics charts (Chart.js)
- 🏆 Leaderboard, achievement badges, activity tracking
- 👤 Editable profiles, support ticket system, FAQ, WhatsApp/Email contact
- 📱 Fully responsive (mobile, tablet, desktop)

## 🛠 Quick Start

### Option A — Automated setup (recommended)

```bash
cd edtech_project
chmod +x setup.sh
./setup.sh
```

This will create a virtual environment, install dependencies, generate & apply migrations, seed demo data, and prompt you to create an admin account.

### Option B — Manual setup

```bash
cd edtech_project

# 1. Create & activate virtual environment
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Configure environment variables
cp .env.example .env
# Edit .env and add your SECRET_KEY, ANTHROPIC_API_KEY, RAZORPAY keys, etc.

# 4. Generate & apply migrations (models are app-ready; migrations are
#    generated locally since they are environment-specific)
python manage.py makemigrations accounts courses tests ai_doubt analytics achievements
python manage.py migrate

# 5. Seed demo data (categories, teachers, sample courses, badges)
python manage.py seed_data

# 6. Create an admin account
python manage.py createsuperuser

# 7. Run the development server
python manage.py runserver
```

Visit **http://127.0.0.1:8000/** 🎉
Admin panel: **http://127.0.0.1:8000/admin/**

> **Note on migrations:** Migration files are intentionally generated by you via `makemigrations` rather than shipped pre-built. This guarantees they exactly match the Django/Python version in your environment and avoids any cross-version migration corruption — this is standard Django practice for any project you receive as source.

## 🔑 Environment Variables (`.env`)

| Variable | Purpose |
|---|---|
| `SECRET_KEY` | Django secret key (generate a new one for production) |
| `DEBUG` | `True` for development, `False` for production |
| `ANTHROPIC_API_KEY` | Powers the AI Doubt Solver (get one at console.anthropic.com) |
| `RAZORPAY_KEY_ID` / `RAZORPAY_KEY_SECRET` | Payment gateway credentials |
| `EMAIL_HOST_USER` / `EMAIL_HOST_PASSWORD` | For password-reset & ticket emails (use a Gmail App Password) |
| `WHATSAPP_NUMBER` / `SUPPORT_EMAIL` | Shown on the Support page |

Generate a secure `SECRET_KEY`:
```bash
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

## 🔒 Security Highlights

- Custom `User` model with **UUID primary keys** (not sequential integers — harder to enumerate)
- Passwords hashed via Django's PBKDF2 (never stored in plain text)
- Django's built-in CSRF protection on every form & AJAX call
- Razorpay payments are **signature-verified server-side** before activating enrollment
- Password-reset tokens are time-limited (1 hour) and single-use
- `SECURE_*` headers, HSTS, and cookie-security flags auto-enabled when `DEBUG=False`
- Environment secrets isolated in `.env` (never commit this file — see `.gitignore`)
- Input validation via Django forms; `bleach` available for sanitizing any rich-text fields you add

**Before deploying to production:**
1. Set `DEBUG=False`
2. Set a strong, unique `SECRET_KEY`
3. Set `ALLOWED_HOSTS` to your real domain
4. Switch `DATABASE_URL` to PostgreSQL
5. Configure a real email backend (not console backend)
6. Put `media/` behind S3 or similar (local disk doesn't scale / persist on most hosts)
7. Run `python manage.py check --deploy` and resolve any warnings

## 🎨 Design System

All design tokens (colors, spacing, shadows, radii, gradients) live in CSS custom properties at the top of `static/css/main.css` — change them once, and the whole UI updates. Dark mode is handled via `[data-theme="dark"]` overrides and toggled client-side (persisted in `localStorage`).
